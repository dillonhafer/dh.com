HowToBackupWithRdiffBackup¶

Primary source of information on this are the online docs, a user-written guide to unattended backups, and the man page.

Sample backup command:

rdiff-backup --verbosity 5 boar-backup::/home /home/rdiff/disk1/backup

Verbosity 5 is used so it logs what files have changed.

Sample restore command:

rdiff-backup --restore-as-of now /home/rdiff/disk1/backup /home/jackc/restore

Note that when backing up from remote machines the SSH permissions may not be such that a direct restore is possible. For example, the backup server may only have permissions to run the backup on the remote server, and the remote server may not have any permissions on the backup server. In this case it would be necessary to temporarily adjust permissions to run a complete restore.

How to clear old history (sudo backup user (or whatever the owner of the backup is)):

rdiff-backup --remove-older-than 2W --force /var/disk3/backup
