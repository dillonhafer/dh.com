SSH Tunneling

Context

You work in an environment of multiple servers/computers. Only one main web/database server is open to a public facing IP address, while the rest are behind internal firewalls. Of course SSH is ONLY allowed using key-pairs.

Problem

You are away from work either on vacation, or just at home for the night, and you need to access information on one of the internal servers. Because the private server does not have a public IP address you cannot directly access it remotely.

Solution

Set up an SSH tunnel to create a bridge from your computer to the private server through the public server.

Action

Set up port tunnel: 
  • ssh -L 2222:privateIPaddress:22 publicIPaddress
Open new terminal

Connect to tunnel: 
  • ssh -P 2222 localhost
